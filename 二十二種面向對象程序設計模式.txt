二十二種面向對象程序設計模式

第一種：Builder

App 實例委托 Builder 實例：

// TextConverter builder = new ASCIITextConverter();
// TextConverter builder = new TeXTextConverter();
TextConverter builder = new TextWidgetConverter();

while ((Token token = tokenIterator.next()) != null) {

  switch (token.type) {

    case CHARACTER:

      builder.convertCharacter(token.character);
      break;

    case FONT_CHANGE:

      builder.convertFontChange(token.fontChange);
      break;

    case PARAGRAPH:

      builder.convertParagraph();
      break;

  }

}

// ASCIIText result = builder.getASCIIText();
// TeXText result = builder.getTeXText();
TextWidget result = builder.getTextWidget();

第二種：Factory

App 實例委托 Factory 實例：

// WidgetFactory factory = new PMWidgetFactory();
WidgetFactory factory = new MotifWidgetFactory();

Window window = factory.createWindow();
ScrollBar scrollBar = factory.createScrollBar();

第三種：Prototype

問題是想讓一個 object 返回一個自己的克隆。辦法是給該 object 對應的 class 增加一個方法，在該方法中調用 constructor 並傳入 this，讓 constructor 將 this 的狀態複製給新創建的 object 然後返回。

第四種：Singleton

問題是想讓一個 class 擁有一組固定的實例。辦法是將這組固定的實例存儲在該 class 的一個靜態字段中，並提供一個靜態方法，用於返回指定的實例。

第五種：Adapter

問題是包裝已有類。辦法是讓包裝類繼承已有類或讓包裝類的實例委托已有類的實例。

第六種：Intermediate Layer

App 實例委托 Window 具體類實例：

// Window window = new IconWindow();
Window window = new TransientWindow();

window.drawBorder();
window.drawCloseButton();

Window 具體類繼承 Window 抽象類：

drawRect();
drawText();

Window 抽象類實例委托 WindowImp 具體類實例：

// WindowImp windowImp = new XWindowImp();
WindowImp windowImp = new PMWindowImp();

windowImp.devDrawLine();
windowImp.devDrawText();

第七種：Composite

問題是想通過同一套 interface 使用單一部件和合成部件。辦法是讓合成部件將請求轉發給它的所有内部部件。

第八種：Decorator

問題是想通過同一套 interface 使用原始類和裝飾類。辦法是讓裝飾類的實例委托原始類的實例。

第九種：Facade

問題是一個系統包含了多個子系統，但是它沒有外觀，想提供一個缺省的外觀。辦法是提供一個缺省的外觀。

第十種：Flyweight

問題是想用一個角色表示多個個體。辦法是將對象和個性狀態解耦。

第十一種：Proxy

問題是想通過同一套 interface 使用實體類和代理類。辦法是讓代理類的實例委托實體類的實例。

第十二種：Chain of Responsibility

問題是想通過同一套 interface 使用所有服務員。辦法是讓所有服務員實現同一套 interface。在服務員的每一個服務方法中，要麽提供服務，要麽把請求轉發給他的直屬領導。

第十三種：Command

MenuItem 實例委托 Command 實例：

Command findCommand = new FindCommand(application);
findCommand.execute();

Command replaceCommand = new ReplaceCommand(application);
replaceCommand.execute();

Command 實例委托 Application 實例：

Document document = application.getDocument();
document.find(what);

Document document = application.getDocument();
document.replace(what, with);

第十四種：Interpreter

問題是想讓用戶使用一種專門的領域語言表示信息。辦法是做一個專門的解釋器。

第十五種：Iterator

問題是想讓用戶以迭代的方式訪問集合元素。辦法是為該集合實現一個迭代器。

第十六種：Mediator

Widget 實例委托 Mediator 實例：

mediator.widgetChanged(this);

Mediator 實例委托 Widget 實例：

listBox.getSelection();
entryField.setText();

第十七種：Memento

問題是想記錄一個對象的狀態的增量變更歷史以實現 undo/redo。辦法是使用一個帶頭節點的雙向鏈表存儲變更日志。

第十八種：Pub/Sub

Subscriber 實例委托 Publisher 實例：

publisher.add(eventType, this);
publisher.remove(eventType, this);

Publisher 實例委托 Subscriber 實例：

subscriber.notice(event);

第十九種：State

TCPConnection 實例委托 TCPState 實例：

// TCPState state = new TCPClosed();
// TCPState state = new TCPListen();
TCPState state = new TCPEstablished();

state.handle();

第二十種：Strategy

Composition 實例委托 Compositor 實例：

// Compositor compositor = new SimpleCompositor();
// Compositor compositor = new TeXCompositor();
Compositor compositor = new ArrayCompositor();

compositor.compose();

第二十一種：Template Method

問題是想讓父類方法能夠調用子類方法。辦法是讓父類定義一些抽象方法，並在其它方法中調用這些抽象方法，讓子類實現這些抽象方法。

第二十二種：Visitor

Node 實例委托 Visitor 實例：

visitor.visitNode(this);
